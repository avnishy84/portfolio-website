Generative AI for Image Synthesis: Virtual Try-On Systems Using Diffusion Models

Avnish Yadav
2025 AI · Software Engineering 3 pages
Abstract

Virtual try-on systems allow users to visualize clothing on their own image without physically trying it on. This paper documents the architecture of an AI-powered virtual try-on application built using Google Gemini Nano for garment description and Banana AI for diffusion model inference. Key topics include the image preprocessing pipeline, prompt engineering for consistent garment rendering, latency management for diffusion model inference, and the UX challenges of presenting variable-quality AI-generated output to users.

Diffusion models have achieved remarkable results in image synthesis tasks, including virtual try-on — the task of generating a realistic image of a person wearing a specific garment. Building a production virtual try-on system requires solving several engineering challenges beyond the model itself: image preprocessing, prompt engineering, latency management, and quality control.

1. System Architecture

The AI Dress Try-On application uses a two-stage pipeline. In the first stage, Google Gemini Nano analyzes the uploaded garment image and generates a structured description including color, texture, style, and fit. This description is used to construct the diffusion model prompt, ensuring that the generated image accurately reflects the garment's characteristics rather than relying on the user to describe it.

In the second stage, the structured garment description and the user's photo are passed to a Banana AI-hosted diffusion model fine-tuned for virtual try-on. The model generates a composite image showing the user wearing the garment, preserving the user's pose and body shape while replacing their clothing.

2. Prompt Engineering for Consistent Rendering

The quality of diffusion model output is highly sensitive to prompt construction. For virtual try-on, the key challenge is ensuring that the generated image preserves the user's identity and pose while accurately rendering the garment. The prompt engineering strategy uses a structured template with fixed sections for identity preservation, pose description, garment description, and quality modifiers.

Negative prompts — instructions for what the model should not generate — are as important as positive prompts for virtual try-on. Common negative prompt elements include "distorted hands", "extra limbs", "blurry face", and "wrong skin tone". These constraints significantly reduce the rate of anatomically incorrect generations.

3. Latency Management

Diffusion model inference is computationally expensive, with cold-start times of 15-30 seconds on Banana AI's serverless GPU infrastructure. The application manages this latency through a combination of optimistic UI (showing a loading animation with progress indication), result caching (storing generated images in Firebase Storage so the same garment/user combination is not regenerated), and warm-up requests (pinging the Banana AI endpoint before the user initiates a generation to reduce cold-start probability).

4. Quality Control and UX

Diffusion model output quality is variable — some generations are photorealistic and accurate, while others have artifacts, distortions, or incorrect garment rendering. The application addresses this through a multi-generation approach: three images are generated for each request, and the user selects the best result. A lightweight quality scoring model (a fine-tuned CLIP model) pre-filters generations with obvious artifacts before presenting them to the user.

The UX design acknowledges the AI's limitations explicitly: the interface labels results as "AI-generated previews" and includes a disclaimer that the actual garment may differ. User research showed that transparent communication about AI limitations increased user trust and reduced negative feedback about imperfect generations.

5. Conclusion

Building production virtual try-on systems requires solving engineering challenges at every layer of the stack: prompt engineering for consistent model output, latency management for acceptable user experience, and quality control for variable AI output. The most important lesson is that the model is only one component of the system — the engineering around the model determines whether the system is usable in practice.

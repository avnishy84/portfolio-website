Building Chrome Extensions with Manifest V3: Architecture, DOM Scraping, and Testing

Avnish Yadav
2024 Software Engineering · Full-Stack 3 pages
Abstract

Chrome's Manifest V3 migration introduced significant architectural constraints that affect how extensions interact with web pages. This paper documents the architecture, DOM scraping strategy, and testing approach developed for TeamChatLogs — a Chrome extension that exports Microsoft Teams chat history. Key topics include MV3's service worker model, virtualized DOM scraping for dynamically rendered content, resilient CSS selector strategies, and a comprehensive 87-test suite combining Jest unit tests with fast-check property-based tests.

Chrome's transition from Manifest V2 to Manifest V3 represents the most significant architectural change to the Chrome extension platform since its introduction. The primary change — replacing persistent background pages with ephemeral service workers — has profound implications for extensions that need to maintain state or perform long-running operations.

1. MV3 Architecture

In MV3, the background script runs as a service worker that can be terminated by the browser at any time when idle. This eliminates the ability to maintain in-memory state between events. The TeamChatLogs extension addresses this by persisting all state to chrome.storage.local and treating each service worker activation as stateless — reading required state from storage at the start of each operation.

2. Virtualized DOM Scraping

Microsoft Teams renders chat messages using a virtualized list — only the messages currently visible in the viewport are present in the DOM. Scraping the complete chat history requires scrolling through the conversation while capturing messages as they enter the DOM, then deduplicating the captured messages.

The scraping algorithm uses a MutationObserver to detect when new message elements are added to the DOM as the user scrolls. Each observed message is parsed and stored in chrome.storage.local with a deduplication key derived from the message timestamp and sender. The scroll automation uses requestAnimationFrame to pace the scroll speed, preventing the Teams virtualization engine from skipping messages.

3. Resilient Selector Strategy

CSS selectors for dynamically rendered applications are inherently fragile — a Teams UI update can change class names and break selectors overnight. The TeamChatLogs extension uses a multi-layer selector strategy: primary selectors target semantic attributes (aria-label, data-tid) that are more stable than class names; fallback selectors use structural patterns (nth-child relationships) as a secondary option; and a health-check mechanism detects when selectors stop matching and alerts the user.

4. Testing Strategy

The 87-test suite is organized into three layers. Unit tests (62 tests) cover the message parsing logic, deduplication algorithm, and export formatters using Jest with jsdom. Integration tests (18 tests) verify the message passing protocol between the content script, service worker, and popup using Chrome extension testing utilities. Property-based tests (7 tests) use fast-check to verify invariants: the deduplication algorithm is idempotent, the export formatter handles all valid message structures, and the scroll algorithm terminates for any finite conversation length.

The property-based tests were particularly valuable for the deduplication algorithm, which had a subtle bug where messages with identical timestamps from different senders were incorrectly deduplicated. The PBT framework found this case within seconds of the first test run by generating messages with the same timestamp but different senders.

5. Conclusion

Building reliable Chrome extensions for dynamically rendered applications requires careful attention to the MV3 architectural constraints, robust DOM scraping strategies that account for virtualization, and a comprehensive testing approach that includes property-based tests for the core algorithms. The TeamChatLogs extension demonstrates that these challenges are tractable with the right architectural decisions.

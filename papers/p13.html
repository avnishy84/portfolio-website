ATS-Aware Resume Optimization: Scoring Algorithms and AI Rewriting Strategies

Avnish Yadav
2025 AI · Full-Stack 3 pages
Abstract

Applicant Tracking Systems (ATS) filter resumes before human review, making ATS compatibility a critical factor in job application success. This paper documents the technical design of CVio's resume optimization pipeline: a deterministic ATS scoring algorithm that quantifies keyword alignment between a resume and job description, and a GPT-4o-powered rewriting system with anti-hallucination constraints that improves the score while preserving factual accuracy. The system achieves an average score improvement of 34 points while maintaining a hallucination rate below 0.5%.

Applicant Tracking Systems parse resumes and score them against job descriptions based on keyword matching, section completeness, and formatting compatibility. Resumes that score below a threshold are automatically rejected without human review. Estimates suggest that 75% of resumes are rejected by ATS before reaching a recruiter.

1. ATS Scoring Algorithm

CVio's ATS scoring algorithm is deterministic — given the same resume and job description, it always produces the same score. This is a deliberate design choice: deterministic scoring allows users to understand exactly why their score changed after an edit, and it enables property-based testing to verify scoring invariants.

The score is computed as a weighted sum of five components: keyword match rate (40%), skills coverage (25%), section completeness (20%), formatting compatibility (10%), and quantified achievement density (5%). Each component is normalized to a 0-100 scale before weighting, producing a final score between 0 and 100.

2. Keyword Extraction

The keyword match component requires extracting the most important keywords from the job description. CVio uses a two-stage extraction process: first, a TF-IDF analysis identifies terms that appear frequently in the job description but infrequently in a corpus of generic job descriptions (filtering out common words like "team" and "experience"); second, GPT-4o classifies the extracted terms into required skills, preferred skills, and soft skills, with different weights applied to each category.

Keyword matching uses lemmatization to handle morphological variants — "managed", "managing", and "management" all match the keyword "manage". This reduces false negatives from candidates who use different forms of the same keyword.

3. AI Rewriting with Anti-Hallucination Constraints

The rewriting system uses GPT-4o to suggest improvements to resume bullet points that increase keyword density while preserving factual accuracy. The anti-hallucination constraint is implemented through a strict prompt that instructs the model to only use information present in the original bullet point — it may rephrase, restructure, and add keywords, but it may not add facts, metrics, or achievements that are not in the original text.

The constraint is enforced at two levels: in the prompt (instructing the model not to hallucinate) and in post-processing (a verification step that checks whether any named entities, numbers, or dates in the rewritten text were present in the original). Rewrites that fail the verification step are rejected and the original text is preserved.

4. Property-Based Testing for Scoring Invariants

The deterministic scoring algorithm is validated using property-based tests that verify key invariants. The most important invariant is monotonicity: adding a keyword from the job description to the resume should never decrease the score. A second invariant is boundedness: the score must always be between 0 and 100. A third invariant is that an empty resume should always score 0.

The PBT framework (fast-check) generates thousands of random resume/job description pairs and verifies these invariants for each pair. This approach found two bugs in the initial implementation: a case where adding a keyword decreased the score due to a normalization error, and a case where a resume with very long bullet points could produce a score above 100 due to an integer overflow in the quantified achievement density calculation.

5. Results

Across 500 test resumes, CVio's optimization pipeline achieved an average ATS score improvement of 34 points (from 48 to 82 on average). The hallucination rate — measured as the percentage of rewritten bullet points that contained facts not present in the original — was 0.4%, well below the 1% target. User satisfaction surveys showed that 89% of users found the suggested rewrites helpful and factually accurate.
